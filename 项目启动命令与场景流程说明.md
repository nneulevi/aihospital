# 项目启动命令与场景流程说明

本文档用于协作人拉取项目后启动完整平台，并按真实业务场景验证患者端、医生端、管理员端、头部 CT AI、报告服务和 EMR 归档链路。

## 1. 启动前提

### 1.1 必要服务

本项目完整链路依赖以下本地服务：

| 服务 | 用途 |
|---|---|
| PostgreSQL + pgvector | Project2 主平台数据库、RAG 向量库、报告/EMR 数据库 |
| Python 环境 | Filter、Lesion、Orchestrator、Report、EMR 服务 |
| Java + Maven Wrapper | Project2 Spring Boot 主平台 |
| Node.js | Vue 前端 |

### 1.2 本地私密配置

这些文件不应上传 Git 仓库，协作者需按本地环境配置：

| 文件 | 用途 |
|---|---|
| `HeadCTOrchestrator/scripts/headct_rag_env.local.ps1` | RAG 数据库、DashScope/百炼 API Key、Embedding、Rerank、LLM 配置 |
| `scripts/project2_env.local.ps1` | Project2 本地数据库账号密码 |

若缺少本地配置，启动脚本会使用默认值，但真实 RAG/LLM 链路可能无法完整工作。

## 2. 完整平台启动命令

在 PowerShell 中执行：

```powershell
cd D:\exam

chcp 65001
$env:PYTHONUTF8 = "1"
$env:PYTHONIOENCODING = "utf-8"

.\scripts\start_headct_platform.ps1 -Restart
```

启动成功时应看到类似输出：


### 2.1 Reload 模式

默认建议使用普通模式，避免 Windows 下 `uvicorn --reload` 导致日志快速膨胀。

如确实需要热更新：

```powershell
cd D:\exam
$env:HEADCT_ALLOW_UVICORN_RELOAD = "true"
.\scripts\start_headct_platform.ps1 -Reload
```

若不设置 `HEADCT_ALLOW_UVICORN_RELOAD=true`，脚本会进入安全 reload 模式：不强制开启 uvicorn reload，但会避免重复手动关闭旧端口。

### 2.2 只启动 AI 子系统，不启动 Project2

```powershell
cd D:\exam
.\scripts\start_headct_platform.ps1 -Restart -SkipProject2
```

## 5. 真实场景流程

### 5.1 患者端流程

目标：模拟患者从登录到完成就诊相关操作。

1. 患者登录。
2. 查看首页个人信息、今日挂号、待缴费、就诊记录。
3. 选择科室、医生、号源，完成预约挂号。
4. 查看挂号记录和排队状态。
5. 查看待缴费项目，确认缴费状态与挂号/检查/处方状态一致。
6. 查看检查检验、处方、报告记录。
7. 使用 AI 问诊，要求 SSE 流式逐字输出，且能结合 RAG 与会话记忆回答。
8. 移动端/小程序患者端应与 Web 患者端数据一致，但交互更适合手机窄屏。

### 5.2 医生端流程

目标：模拟医生接诊、开立项目、查看 AI 结果、生成诊断建议。

1. 医生登录。
2. 查看待接诊患者队列。
3. 选择患者并开始接诊。
4. 查看患者病历、历史记录、检查结果和处方记录。
5. 填写病历：主诉、现病史、既往史、体格检查、初步诊断。
6. 开立检查、检验、处置项目。
7. 开立处方。
8. 使用 AI 辅助诊断，要求：
   - SSE 流式逐字输出；
   - 结合 RAG 医学知识；
   - 能记住同一会话上下文；
   - 输出自然语言，不展示底层 JSON。
9. 查看头部 CT AI 分析结果、报告状态和 EMR 归档结果。

### 5.3 管理员端流程

目标：验证医院业务管理数据完整、字段不空、状态一致。

1. 管理员登录。
2. 查看首页统计：挂号、医生、科室、药品、财务等数据。
3. 管理科室与员工。
4. 维护医生排班和号源。
5. 查看财务记录、收费记录。
6. 查看药品库存，字段应包含：
   - 药品编码；
   - 药品名称；
   - 规格；
   - 单位；
   - 价格；
   - 厂家；
   - 库存；
   - 库存预警状态。
7. 验证管理员端显示的数据必须来自后端接口，不应出现前端占位数据。

### 5.4 医技人员流程

目标：验证检查/检验/处置任务独立工作流。

1. 医技人员查看已收费的检查、检验、处置任务。
2. 接收任务并执行。
3. 录入检查/检验/处置结果。
4. 医生端能看到医技结果。
5. 患者端能看到对应报告或记录。
6. 状态应按真实顺序流转：

```text
CREATED -> CHARGED -> EXECUTING -> REPORTED
```

### 5.5 药房流程

目标：验证处方、发药、退药、库存联动。

1. 医生开立处方。
2. 患者或收费员完成处方收费。
3. 药房查看待发药处方。
4. 药师发药，库存减少。
5. 若退药，库存恢复。
6. 库存流水应包含入库、盘点、发药、退药记录。

## 6. 头部 CT AI 场景流程

目标：使用真实影像文件走完整链路，而不是 smoke。

```text
上传 CT 影像
  -> Filter 金属伪影识别/质控
  -> 金属伪影校正或校正策略说明
  -> Lesion 病灶识别
  -> Orchestrator 汇总结构化结果
  -> RAG + LLM 生成自然语言辅助结论
  -> Report 服务生成报告草稿
  -> 医生审核/签署/发布
  -> EMR 服务归档
  -> Project2 医生端/患者端查看结果
```

### 6.1 当前模型状态

| 能力 | 当前状态 |
|---|---|
| 金属伪影识别 | 使用本地训练 3D U-Net 或成熟候选权重，启动脚本避免 smoke 优先 |
| 金属伪影校正 | 已登记 InDuDoNet 成熟 MAR 权重 |
| 颅内出血分类 | 已接入 VinBigData MIDL2020 CNN-LSTM ICH 真实公开权重 `best_resnet50.pth` |
| 颅内出血分割 | 尚未接入成熟公开分割 checkpoint，需后续接 INSTANCE/PhysioNet CT-ICH 等数据 |
| 阈值校准 | 已支持 CQ500 官方 `reads.csv` 监督校准；当前本地子集无官方标签时只允许生成分布参考阈值 |

### 6.2 真实影像推理验证命令

```powershell
cd D:\exam
$env:PYTHONIOENCODING = "utf-8"

python scripts\validate_true_checkpoints.py --strict-mature
```

验证 CQ500 子集转换：

```powershell
python HeadCTLesionDetection\models\hemorrhage\prepare_cq500ct200.py `
  --dicom-root D:\Data `
  --output-dir HeadCTLesionDetection\datasets\cq500ct200_subset\images `
  --manifest HeadCTLesionDetection\datasets\cq500ct200_subset\manifest.csv `
  --limit 12
```

使用 VinBigData 真实权重推理：

```powershell
python HeadCTLesionDetection\models\hemorrhage\run_manifest_inference.py `
  --manifest HeadCTLesionDetection\datasets\cq500ct200_subset\manifest.csv `
  --checkpoint HeadCTLesionDetection\models\hemorrhage\external_weights\best_resnet50.pth `
  --output-jsonl HeadCTLesionDetection\models\hemorrhage\runs\vinbigdata_cq500_calibration\predictions.jsonl `
  --device cpu `
  --threshold 0.5 `
  --image-size 224 `
  --max-slices 48 `
  --sampling-offsets 0 `
  --limit 6
```

生成无监督分布阈值，仅作参考：

```powershell
python HeadCTLesionDetection\models\hemorrhage\calibrate_predictions.py `
  --predictions HeadCTLesionDetection\models\hemorrhage\runs\vinbigdata_cq500_calibration\predictions.jsonl `
  --output HeadCTLesionDetection\models\hemorrhage\runs\vinbigdata_cq500_calibration\calibration.json `
  --mode unsupervised `
  --percentile 95
```

注意：无监督阈值不能作为模型准确率证据，启动脚本默认不会使用该阈值。若要临时启用：

```powershell
$env:VINBIGDATA_ALLOW_UNSUPERVISED_CALIBRATION = "true"
```

### 6.3 接入 CQ500 官方 reads.csv 后的监督评估

若获得官方 `reads.csv`：

```powershell
python HeadCTLesionDetection\models\hemorrhage\prepare_cq500_reads_labels.py `
  --manifest HeadCTLesionDetection\datasets\cq500ct200_subset\manifest.csv `
  --reads-csv <CQ500官方reads.csv路径> `
  --output-labels-csv HeadCTLesionDetection\datasets\cq500ct200_subset\labels.csv
```

运行监督评估与阈值校准：

```powershell
python HeadCTLesionDetection\models\hemorrhage\evaluate.py `
  --labels-csv HeadCTLesionDetection\datasets\cq500ct200_subset\labels.csv `
  --checkpoint HeadCTLesionDetection\models\hemorrhage\external_weights\best_resnet50.pth `
  --provider vinbigdata `
  --device cpu `
  --output-dir HeadCTLesionDetection\models\hemorrhage\runs\vinbigdata_cq500_supervised_eval `
  --threshold 0.5 `
  --calibrate-threshold `
  --vinbigdata-image-size 224 `
  --vinbigdata-max-slices 48
```

监督评估通过后，再将监督校准文件作为运行时校准依据。

## 8. 用户视角验收重点

验收时不要只看接口是否返回 200，应按用户视角确认：

- 页面没有红色错误提示；
- 没有裸露 JSON；
- 没有开发调试字段；
- 中文不乱码；
- 数据不互相矛盾；
- 患者端、医生端、管理员端状态一致；
- 医生审核后的报告能被患者/EMR 正确看到；
- AI 问诊和 AI 辅助诊断是 SSE 逐字输出；
- RAG 能召回头部 CT 医学知识；
- AI 影像结果有影像可视化、置信度、建议与局限性；
- 低置信度时系统应提示复核，而不是强行给确定结论。

## 9. 常见问题

### 9.1 登录失败

先检查 Project2 健康状态：

```powershell
Invoke-RestMethod http://127.0.0.1:8092/actuator/health
```

若数据库 DOWN，检查 `scripts/project2_env.local.ps1` 和本地 PostgreSQL。

### 9.2 AI 问诊没有真实回答

检查：

```powershell
Get-Content HeadCTOrchestrator\scripts\headct_rag_env.local.ps1
```

确认 DashScope/百炼 API Key、Embedding、Rerank、LLM 配置存在。

### 9.3 头部 CT AI 使用 fallback

执行：

```powershell
python scripts\validate_true_checkpoints.py --strict-mature
```

重点查看：

- `lesion_hemorrhage_classification.selection.fallback_used`
- `filter_metal_artifact_reduction.selection.fallback_used`
- `filter_metal_artifact_segmentation.selection.selected_provenance`

### 9.4 中文乱码

所有 PowerShell 运行前建议执行：

```powershell
chcp 65001
$env:PYTHONUTF8 = "1"
$env:PYTHONIOENCODING = "utf-8"
```

脚本与文档统一使用 UTF-8。
