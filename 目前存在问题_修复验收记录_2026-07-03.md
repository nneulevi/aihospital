# 目前存在问题修复与验收记录（2026-07-03）

## 修复范围

1. 患者端门诊挂号
   - 修复 AI 问诊跳转挂号时硬编码医生信息的问题。
   - 现在路由中的 `deptId / doctorId / visitDate / noon` 会回查真实后端号源，未查到号源时停留在医生选择页并提示。
   - 修复确认挂号提交参数不符合后端校验的问题：性别提交为 `M/F`，家庭住址不再为空。
   - 医生剩余号源为 0 时前端不可继续选择。

2. 后端挂号业务一致性
   - 挂号时按同一医生、科室、日期、午别计算已用号源。
   - 号满时阻止继续挂号。
   - 同一患者同一医生同一时段重复挂号会被阻止。
   - 排队号改为该医生该时段已挂号人数 + 1。

3. 智能问诊推荐
   - 移除患者端硬编码推荐医生。
   - 推荐医生改为根据 AI 推荐科室查询未来 7 天真实号源。
   - AI 流式自然语言结论会保留到结果页。
   - 推荐科室理由分条展示，避免只显示浅显一句话。
   - 后端 `ConsultationResponseVO` 增加 `diagnosisHint`，普通接口和流式接口均返回。

4. 检查/检验预约
   - 检查、检验项目支持选中态。
   - 提交时绑定患者最新有效挂号记录。
   - 真实调用患者端检查/检验申请接口，提交后刷新“我的申请”。
   - 没有有效挂号时给出明确提示。

5. 管理员号源查询
   - 号源查询改为统一 `{ query: ... }` 参数封装。
   - 查询结果兼容统一响应结构。

## 2026-07-04 追加修复：AI 排班历史查询空白浮层

问题现象：

- 管理员端点击“查询历史排班”后，按钮区域上方出现无上下文的白色浮层，用户感知像是空白提示或按钮无效。

根因：

- `AI生成排班` 与 `查询历史排班` 使用全局 Toast 展示成功/空数据状态。
- 在当前移动端排班页面中，Toast 会覆盖按钮区域；连续点击“AI生成排班 -> 查询历史排班”时，上一条成功 Toast 还可能残留，造成空白遮挡感。

修复：

- `frontend/src/views/admin/AISchedule.vue`
  - 增加页面内 `history-feedback` 状态条。
  - `AI生成排班`、`查询历史排班` 的查询中、成功、空数据、失败状态均在页面内展示。
  - 历史查询成功时直接显示“已查询到 N 条历史排班”或“当前筛选条件下暂无历史排班”，不再依赖全局 Toast。
