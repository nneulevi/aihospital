# 用户级审查与修复记录（2026-07-01）

## 审查目标

以真实用户视角检查系统，而不是只验证接口是否返回成功：

- 页面信息是否可读、专业、清晰。
- 交互是否顺畅，是否存在空状态误显示、裸 JSON、红色错误提示。
- 前端、Project2 主平台、头部 CT AI、报告服务、EMR 归档链路是否能贯通。
- 截图证据是否能真实反映用户看到的页面。

## 本次发现并修复的问题

### 1. 报告工作台 AI 分析默认展示原始 JSON

问题表现：

- 医生进入“头部 CT 检查报告”工作台后，“AI 分析”页签直接显示 `quality_control`、`lesion_analysis` 等原始 JSON。
- 对非开发者用户不友好，无法快速理解 AI 给出的辅助价值。

修复：

- 将页签名称从“AI 分析”调整为“AI 解读”。
- 默认展示自然语言结构化内容：
  - 图像质量
  - 伪影比例
  - 病灶识别状态
  - 模型版本
  - AI 辅助结论
  - 病灶阳性置信度条
  - 建议医生重点复核
  - 局限性与注意事项
  - 影像可视化输出
- 原始 JSON 保留在折叠的“技术详情：原始 AI 结果”中，便于需要时追溯，但不再默认暴露给业务用户。

涉及文件：

- `HeadCTReportService/frontend/index.html`
- `HeadCTReportService/frontend/app.js`
- `HeadCTReportService/frontend/styles.css`

### 2. 病灶置信度字段缺失时，页面直接不显示置信度区域

问题表现：

- 当病灶模型没有返回 `positive_probability` 等字段时，“病灶阳性置信度”整块缺失。
- 用户会误解为页面漏渲染或 AI 结果不完整。

修复：

- 置信度条始终展示。
- 有数值时显示百分比；无数值时明确显示“未返回”。
- 保证页面结构稳定，医生能知道是模型未返回，而不是页面坏了。

### 3. 模型版本出现 `[object Object]`

问题表现：

- 报告工作台的模型版本卡片会把对象直接渲染成 `[object Object]`。
- 对医生无意义，且破坏页面专业性。

修复：

- 增加模型版本格式化逻辑。
- 对嵌套对象优先展示 `version`、`model_version`、`checkpoint`、`name`、`backend` 等可读字段。

### 4. 报告页空状态与正文可能同时显示

问题表现：

- `hidden` 属性被 `.empty-state { display: grid }` 覆盖。
- 严格截图验收时发现最终归档截图可能仍显示“请选择一份检查报告”。

修复：

- 增加 `[hidden] { display: none !important; }`，确保隐藏状态不会被组件样式覆盖。
- 收紧 Playwright 断言：最终归档截图前必须确认报告内容可见、空状态隐藏、EMR 编号可见。

涉及文件：

- `HeadCTReportService/frontend/styles.css`
- `frontend/tests/visual/headct-real-user-workflow.spec.ts`
