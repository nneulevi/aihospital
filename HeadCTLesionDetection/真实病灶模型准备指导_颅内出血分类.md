# 真实病灶模型准备指导：颅内出血分类

## 1. 阶段目标

当前 `HeadCTLesionDetection` 已完成 mock 合同服务，能够返回固定阴性病灶识别结果，并已被 `HeadCTOrchestrator` 编排调用。

下一步建议准备第一个真实病灶模型：

```text
颅内出血 series/study 级分类模型
```

第一阶段目标不是做分割，也不是直接生成诊断报告，而是完成：

```text
输入一例头部 CT
  -> 输出是否疑似颅内出血
  -> 输出置信度
  -> 输出报告辅助提示
  -> 能被 HeadCTLesionDetection 服务调用
  -> 能被 HeadCTOrchestrator 汇总
```

## 2. 为什么先做颅内出血分类

颅内出血适合作为第一个真实病灶模型，原因：

- 急诊价值高。
- 头部 CT 中相对常见。
- 可先做 series/study 级分类，不需要马上做像素级分割。
- 后续可以自然扩展到 slice 级定位、出血区域分割和出血亚型分类。

不建议第一阶段直接做：

- 出血区域分割。
- 出血亚型多分类。
- 脑梗死早期识别。
- 中线移位结构测量。

这些任务对标注质量、模型验证和临床解释要求更高。

## 4. 第一阶段任务定义

### 4.1 输入

第一阶段输入：

```text
.nii
.nii.gz
```

体数据维度：

```text
Z x H x W
```

### 4.2 输出

模型输出：

```json
{
  "lesion_type": "intracranial_hemorrhage",
  "detected": true,
  "confidence": 0.91,
  "severity": "suspected",
  "affected_slices": [],
  "model_name": "head_ct_hemorrhage_classifier",
  "model_version": "v1.0.0",
  "report_suggestion": "疑似颅内出血，请医生结合原始影像复核。"
}
```

第一阶段 `affected_slices` 可以为空，因为 series/study 级分类不负责定位。

### 4.3 标签

二分类标签：

```text
0: no_hemorrhage
1: hemorrhage
```

如果数据集中有多个出血亚型，第一阶段先合并：

```text
epidural
intraparenchymal
intraventricular
subarachnoid
subdural
```

只要任一亚型为阳性，就标记：

```text
hemorrhage = 1
```

## 5. 数据准备

### 5.1 推荐数据来源

当前项目已有 CQ500 相关文档，可优先研究 CQ500 标签字段。具体数据格式参见目录下CQ500T数据集字段释义.md及 CQ500 颅脑CT急诊数据集详细介绍(1).md

需要明确：

- 哪个文件记录 CT 序列。
- 哪个文件记录出血标签。
- 标签是 patient 级、study 级、series 级还是 slice 级。
- 是否存在多位医生读取结果。
- 标签是否需要聚合。

### 5.2 数据目录建议

建议整理为：

```text
HeadCTLesionDetection/
  datasets/
    hemorrhage/
      images/
        case_0001.nii.gz
        case_0002.nii.gz
      labels.csv
```

`labels.csv` 推荐字段：

```csv
case_id,image_path,hemorrhage,source,split
case_0001,images/case_0001.nii.gz,1,CQ500,train
case_0002,images/case_0002.nii.gz,0,CQ500,val
```

可选字段：

```csv
patient_id,study_id,series_id
epidural,intraparenchymal,intraventricular,subarachnoid,subdural
```

### 5.3 数据划分

建议：

```text
train: 70%
val:   15%
test:  15%
```

注意：

- 必须按 patient/study 级划分，避免同一患者不同序列同时进入训练和测试。
- 不要按 slice 随机切分。
- 如果数据很少，先固定 val/test，避免频繁改动测试集。

## 6. 预处理建议

### 6.1 HU 裁剪

脑窗建议：

```text
window center: 40
window width: 80
```

等价裁剪：

```text
[-40, 120]
```

可另加宽窗作为多通道输入：

```text
brain window: [-40, 120]
blood/subdural window: [0, 200]
bone window: [-1000, 1000]  # 第一阶段可不加
```

第一版建议先做单通道脑窗，降低复杂度。

### 6.2 尺寸归一化

建议把体数据重采样或 resize 到统一尺寸。

第一阶段可选：

```text
Z: 32 或 64
H: 224
W: 224
```

例如：

```text
input_shape = [32, 224, 224]
```

如果显存不足：

```text
input_shape = [32, 160, 160]
```

### 6.3 强度归一化

裁剪后归一化到：

```text
[0, 1]
```

或：

```text
mean/std 标准化
```

建议第一版使用 `[0,1]`，便于复现。

## 7. 模型方案

### 7.1 推荐第一版模型

第一版推荐：

```text
2.5D / 3D CNN 分类器
```

可选方案：

1. 3D ResNet-like classifier。
2. 轻量 3D CNN。
3. 2D slice encoder + pooling。

不建议第一版直接使用复杂 Transformer。

### 7.2 轻量 3D CNN 示例结构

```text
Input: [B, 1, Z, H, W]
Conv3D + BN + ReLU + Pool
Conv3D + BN + ReLU + Pool
Conv3D + BN + ReLU + Pool
Global Average Pooling
Linear
Sigmoid
```

输出：

```text
logit: [B, 1]
```

损失：

```text
BCEWithLogitsLoss
```

如果类别不平衡：

```text
pos_weight = negative_count / positive_count
```

## 8. 训练配置建议

建议默认超参数：

```text
batch_size: 2~8
epochs: 30
lr: 1e-4
weight_decay: 1e-4
optimizer: AdamW
scheduler: CosineAnnealingLR
loss: BCEWithLogitsLoss
metric: AUC, sensitivity, specificity, accuracy
```

如果 GPU 显存有限：

```text
batch_size: 1~2
gradient_accumulation: 4
input_shape: [32, 160, 160]
```

## 9. 评估指标

至少输出：

```text
AUC
accuracy
sensitivity / recall
specificity
precision
F1
confusion matrix
```

医疗场景中，颅内出血更关注：

```text
sensitivity
negative predictive value
```

但不能只看 sensitivity，也要关注 false positive。

建议保存：

```text
runs/hemorrhage_v1/
  metrics.json
  confusion_matrix.png
  roc_curve.png
  pr_curve.png
```

## 11. 与 mock 服务的替换方式

当前 mock 服务中有：

```text
mock_lesion_result()
```

真实模型接入后，不要删除 mock。

建议改为：

```text
LESION_MODE=mock
LESION_MODE=model
```

当：

```text
LESION_MODE=mock
```

返回固定阴性结果。

当：

```text
LESION_MODE=model
```

调用：

```python
predict_hemorrhage(...)
```

这样 mock 可继续用于接口测试。

## 12. 质量上下文 quality_context 的使用

从 Orchestrator 传入：

```json
{
  "artifact_detected": true,
  "artifact_ratio": 0.034,
  "severity": "moderate",
  "affected_slices": [42, 43, 44],
  "analysis_reliability": "limited_by_artifact"
}
```

真实模型可根据它调整：

- `reliability`
- `warnings`
- `report_suggestion`

但第一版不要让它直接改变模型概率。

建议：

```text
模型负责输出概率
quality_context 负责解释可信度
```

## 13. 服务接入验收

真实模型接入 `HeadCTLesionDetection` 后，必须满足：

- `/api/head-ct-lesion/health` 能显示模型加载状态。
- `/api/head-ct-lesion/tasks` 能触发真实推理。
- `/api/head-ct-lesion/results/{task_id}` 返回真实模型结果。
- mock 模式仍可用。
- Orchestrator 在 `LESION_SERVICE_ENABLED=true` 时能汇总真实结果。
- smoke test 覆盖 mock 模式和 model 模式。

## 14. 最小可用版本验收标准

第一版真实颅内出血模型至少应达到：

```text
能加载 checkpoint
能对一例 .nii.gz 输出 confidence
能写入 lesion_result.json
能被 Orchestrator 汇总
能输出 metrics.json
```

不要求第一版达到临床可用，但必须明确：

```text
研究/演示模型，不作为最终诊断依据
```

## 15. 风险与注意事项

### 15.1 标签风险

CQ500 等公开数据可能存在：

- 标签粒度不一致。
- 多医生意见不一致。
- 标签并非精确到 slice。
- 部分病例质量不佳。

必须在 README 中说明标签来源和聚合规则。

### 15.2 数据泄漏风险

避免：

```text
同一患者 / 同一 study 的不同序列同时出现在 train 和 test
```

### 15.3 医疗表述风险

模型输出只能写：

```text
疑似
建议复核
未见明确征象
供医生参考
```

不要写：

```text
确诊
排除
无需医生审核
```

## 18. nnU-Net v2 分割训练补充

nnU-Net 是分割路线，不是当前 series/study 级二分类路线。只有具备 voxel 级出血区域 mask 后，才适合进入本流程。

默认工作目录：

```text
D:\exam\nnUNet_raw
D:\exam\nnUNet_preprocessed
D:\exam\nnUNet_results
```

查看并创建工作目录：

```powershell
python HeadCTLesionDetection/models/hemorrhage/train_nnunet.py env
```

准备 nnU-Net 风格数据：

```powershell
python HeadCTLesionDetection/models/hemorrhage/segmentation_prepare.py `
  --labels-csv HeadCTLesionDetection/datasets/hemorrhage/labels.csv `
  --mask-dir HeadCTLesionDetection/datasets/hemorrhage/masks `
  --output-dir D:\exam\nnUNet_raw\Dataset501_Hemorrhage `
  --dataset-name Dataset501_Hemorrhage
```

规划和预处理：

```powershell
python HeadCTLesionDetection/models/hemorrhage/train_nnunet.py plan `
  -d 501 `
  --verify-dataset-integrity `
  --configurations 3d_fullres
```

训练 fold 0：

```powershell
python HeadCTLesionDetection/models/hemorrhage/train_nnunet.py train `
  -d 501 `
  -c 3d_fullres `
  -f 0 `
  --device cuda
```

预测：

```powershell
python HeadCTLesionDetection/models/hemorrhage/train_nnunet.py predict `
  -i D:\exam\nnUNet_raw\Dataset501_Hemorrhage\imagesTs `
  -o D:\exam\nnUNet_results\Dataset501_Hemorrhage\predictionsTs `
  -d 501 `
  -c 3d_fullres `
  -f 0 `
  --device cuda
```

如果只是检查命令，不启动耗时任务，可添加：

```text
--dry-run
```

当前 `requirements-medimg.txt` 已固定 Windows + Python 3.9 可用的 nnU-Net 依赖组合。
