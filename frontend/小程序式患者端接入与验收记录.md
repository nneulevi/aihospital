# 小程序式患者端接入与验收记录

更新时间：2026-06-27 18:04

## 目标

在不破坏原有患者 Web 端 `/patient/*` 的前提下，新增一个独立的小程序式患者端入口 `/mini-patient`，功能要求与患者 Web 端一致，覆盖 AI 问诊、预约挂号、电子病历、门诊缴费、个人中心。

## 接入方式

- 新增入口：`/mini-patient`
- 权限角色：复用 `PATIENT`
- 登录入口：继续复用 `/patient/login`，未登录访问 `/mini-patient/*` 时带 `redirect` 回跳
- 数据来源：复用现有 `frontend/src/api/index.ts` 患者端 API
- 原 Web 端：未修改 `frontend/src/views/patient/*.vue`

## 新增文件

- `frontend/src/views/mini-patient/MiniPatientLayout.vue`
- `frontend/src/views/mini-patient/Home.vue`
- `frontend/src/views/mini-patient/AI.vue`
- `frontend/src/views/mini-patient/Appointment.vue`
- `frontend/src/views/mini-patient/Records.vue`
- `frontend/src/views/mini-patient/Orders.vue`
- `frontend/src/views/mini-patient/Profile.vue`
- `frontend/tests/visual/mini-patient.spec.ts`

## 必要扩展

- `frontend/src/router/index.ts`
  - 仅追加 `miniPatientRoutes`
  - 不替换、不删除、不改写原 `/patient` 路由组

## 功能覆盖

| 功能 | 小程序端路径 | 数据来源 | 状态 |
| --- | --- | --- | --- |
| 首页概览 | `/mini-patient` | `/api/patient/dashboard/summary` | 已接入 |
| AI 智能问诊 | `/mini-patient/ai` | `/api/ai/triage` | 已接入 |
| 预约挂号 | `/mini-patient/appointment` | `/api/patient/doctors`、`/api/patient/register` | 已接入 |
| 电子病历 | `/mini-patient/records` | `/api/patient/records` | 已接入 |
| 门诊缴费 | `/mini-patient/orders` | `/api/patient/orders` | 已接入 |
| 个人中心 | `/mini-patient/profile` | 本地登录态、患者数据 | 已接入 |
