# 前后端功能映射核查报告

## 核查结论

以当前运行中的 Project2 OpenAPI `http://127.0.0.1:8092/v3/api-docs` 为后端接口来源，本次核查到 33 个后端接口。

结论：

- 33 / 33 个后端接口均已在前端存在映射。
- 查询型、分析型、报告型接口均有前端结果展示区域。
- 动作型接口均有前端触发入口，并通过 Toast、状态刷新、列表刷新或页面跳转反馈结果。
- HeadCT AI 子系统不直接由前端逐个调用，而是通过 Project2 的 AI API 统一接入；对应前端入口集中在 AI问诊、AI辅助诊断、AI影像识别、AI报告生成页面流程中。

## 本次补齐内容

### 1. 医生接诊

后端接口：

- `PUT /api/doctor/patients/{registerId}/receive`

前端映射：

- `src/api/index.ts`
- `src/views/doctor/DoctorDashboard.vue`

说明：

- 在医生待诊队列点击患者卡片时，先调用接诊接口。
- 接诊成功后进入患者诊疗页。
- 接诊过程中卡片状态显示为“接诊中”。

### 2. 医生患者详情

后端接口：

- `GET /api/doctor/patients/{registerId}`

前端映射：

- `src/views/doctor/PatientVisit.vue`

说明：

- 就诊页进入后通过 `registerId` 拉取真实患者详情。
- 页面顶部展示姓名、性别、年龄、病历号、挂号 ID。

### 3. AI 排班历史结果

后端接口：

- `GET /api/ai/schedule/result`

前端映射：

- `src/views/admin/AISchedule.vue`

说明：

- AI排班页新增“查询历史排班”入口。
- 历史排班以医生、日期、班次、号源数量展示。

### 4. 患者取消挂号

后端接口：

- `PUT /api/patient/register/cancel`

前端映射：

- `src/views/patient/Records.vue`

说明：

- 确认后调用后端取消接口，并刷新列表。

### 5. 药房通用库存查询

后端接口：

- `GET /api/drugstore/inventory`

前端映射：

- `src/views/admin/DrugManage.vue`

说明：

- 药品库存页新增“药房库存”查询入口。
- 返回结果复用现有库存卡片展示。
