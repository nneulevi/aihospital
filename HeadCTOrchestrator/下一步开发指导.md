## 2. 当前尚未完成的关键点

病灶模型接入链路已经具备，但真实病灶模型本身还未完成训练。

当前缺少：

- `HeadCTLesionDetection/datasets/hemorrhage/labels.csv`
- `HeadCTLesionDetection/datasets/hemorrhage/images/*.nii.gz`
- 如需 nnU-Net 分割训练，还缺少 voxel 级病灶 mask：`HeadCTLesionDetection/datasets/hemorrhage/masks/*.nii.gz`
- 正式训练得到的 `best.pt`
- 正式验证得到的 `metrics.json`
- 测试集推理结果 `predictions.csv`
- 三服务真实模型模式联调记录

因此当前状态应定义为：

```text
服务链路已完成，真实模型接入口已完成，真实 checkpoint 待训练生成。
```

## 3. 下一阶段总目标

下一阶段目标不是继续扩展更多病灶类型，而是先完成第一个真实病灶模型：

```text
颅内出血 series/study 级二分类模型
```

目标链路：

```text
.nii/.nii.gz 头部 CT
  -> Filter 质控
  -> HeadCTLesionDetection 颅内出血分类
  -> Orchestrator 汇总
  -> 返回 quality_control + lesion_analysis + report_assist
```

## 4. P0：准备真实训练数据

### 4.1 目标目录

```text
HeadCTLesionDetection/
  datasets/
    hemorrhage/
      images/
        case_0001.nii.gz
        case_0002.nii.gz
      labels.csv
```

### 4.2 labels.csv 推荐字段

```csv
case_id,image_path,hemorrhage,source,split,patient_id,study_id,series_id
case_0001,images/case_0001.nii.gz,1,CQ500,train,patient_001,study_001,series_001
case_0002,images/case_0002.nii.gz,0,CQ500,val,patient_002,study_002,series_002
```

必需字段：

- `case_id`
- `image_path`
- `hemorrhage`
- `split`

推荐字段：

- `source`
- `patient_id`
- `study_id`
- `series_id`

### 4.3 标注聚合规则

第一阶段只做二分类：

```text
0 = no_hemorrhage
1 = hemorrhage
```

如果数据集中存在出血亚型字段，可以按如下规则聚合：

```text
epidural
intraparenchymal
intraventricular
subarachnoid
subdural
```

任一亚型为阳性，则：

```text
hemorrhage = 1
```

### 4.4 数据划分原则

建议：

```text
train: 70%
val:   15%
test:  15%
```

要求：

- 必须按 patient/study 级别划分。
- 不要把同一患者或同一 study 的不同序列同时放入 train 和 test。
- 不要按 slice 随机划分。
- test 集固定后不要频繁改动。

## 5. P1：训练第一个真实模型

### 5.1 轻量 PyTorch 训练

```powershell
python HeadCTLesionDetection/models/hemorrhage/train.py `
  --labels-csv HeadCTLesionDetection/datasets/hemorrhage/labels.csv `
  --output-dir HeadCTLesionDetection/models/hemorrhage/runs/hemorrhage_v1 `
  --input-shape 32,160,160 `
  --batch-size 2 `
  --device auto
```

预期输出：

```text
HeadCTLesionDetection/models/hemorrhage/runs/hemorrhage_v1/
  best.pt
  metrics.json
  label_map.json
  train_config.json
```

### 5.2 MONAI 规范训练

如果优先走更规范的医学影像训练流程：

```powershell
python HeadCTLesionDetection/models/hemorrhage/train_monai.py `
  --labels-csv HeadCTLesionDetection/datasets/hemorrhage/labels.csv `
  --output-dir HeadCTLesionDetection/models/hemorrhage/runs/monai_densenet121 `
  --network densenet121 `
  --input-shape 32,160,160 `
  --batch-size 2 `
  --device auto
```

说明：

- 当前 `monai` 已安装可用。
- 当前 `nnunetv2` 已安装可用，并已补充 `train_nnunet.py` 训练封装。
- nnU-Net 只有在具备 voxel 级病灶 mask 后才适合进入分割阶段；分类标签不能直接用于分割训练。

### 5.3 nnU-Net v2 分割训练

nnU-Net 是分割训练路线，不是当前颅内出血二分类 checkpoint 的替代品。只有当具备 voxel 级出血区域 mask 时，才进入本流程。

先准备 nnU-Net 工作目录和查看环境变量：

```powershell
python HeadCTLesionDetection/models/hemorrhage/train_nnunet.py env
```

准备数据：

```powershell
python HeadCTLesionDetection/models/hemorrhage/segmentation_prepare.py `
  --labels-csv HeadCTLesionDetection/datasets/hemorrhage/labels.csv `
  --mask-dir HeadCTLesionDetection/datasets/hemorrhage/masks `
  --output-dir D:\exam\nnUNet_raw\Dataset501_Hemorrhage `
  --dataset-name Dataset501_Hemorrhage
```

规划和预处理：

```powershell
python HeadCTLesionDetection/models/hemorrhage/train_nnunet.py plan `
  -d 501 `
  --verify-dataset-integrity `
  --configurations 3d_fullres
```

训练：

```powershell
python HeadCTLesionDetection/models/hemorrhage/train_nnunet.py train `
  -d 501 `
  -c 3d_fullres `
  -f 0 `
  --device cuda
```

预测：

```powershell
python HeadCTLesionDetection/models/hemorrhage/train_nnunet.py predict `
  -i D:\exam\nnUNet_raw\Dataset501_Hemorrhage\imagesTs `
  -o D:\exam\nnUNet_results\Dataset501_Hemorrhage\predictionsTs `
  -d 501 `
  -c 3d_fullres `
  -f 0 `
  --device cuda
```

长耗时任务前可加 `--dry-run` 查看将执行的 nnU-Net 命令。

## 6. P2：模型评估

训练完成后，在 test split 上评估：

```powershell
python HeadCTLesionDetection/models/hemorrhage/evaluate.py `
  --labels-csv HeadCTLesionDetection/datasets/hemorrhage/labels.csv `
  --checkpoint HeadCTLesionDetection/models/hemorrhage/runs/hemorrhage_v1/best.pt `
  --split test `
  --device auto
```

至少输出：

```text
runs/hemorrhage_v1/eval/
  metrics.json
  predictions.csv
```

重点关注指标：

- AUC
- sensitivity / recall
- specificity
- precision
- F1
- accuracy
- confusion matrix

医学场景中，颅内出血更关注 sensitivity 和 negative predictive value，但不能只追求召回率，也要控制 false positive。

## 9. P5：完善报告辅助与医生审核闭环

模型稳定后再进入本阶段。

建议新增：

```text
POST /api/head-ct-ai/reviews/{task_id}
GET  /api/head-ct-ai/reviews/{task_id}
```

审核记录结构：

```json
{
  "task_id": "xxx",
  "review_status": "confirmed",
  "doctor_id": "doctor_001",
  "reviewed_at": "2026-06-11T10:30:00",
  "doctor_comment": "已结合原始影像复核。",
  "final_report_used": true
}
```

原则：

- AI 结果只作为辅助参考。
- 不输出“确诊”“排除”“无需复核”等绝对表述。
- 报告辅助文本应始终保留医生复核提示。

## 11. 下一次开发建议

阶段 J 已完成，下一步优先进入阶段 K：

1. 设计门诊就诊、患者、医生与检查报告之间的统一业务标识。
2. 新建独立的门诊/AI 助理医生模块，不将门诊业务写入 Orchestrator。
3. 仅消费 `HeadCTReportService` 中已签署或已发布的报告。
4. 建立患者病历摘要、检查结果时间线和医生工作列表。
5. 为 AI 助理医生定义结构化输入、引用来源、医生确认和审计契约。
6. 禁止 AI 助理医生自动修改电子病历、开立处方或替代医生下达诊疗决定。

真实病灶模型仍作为并行工作继续推进：

1. 整理 `HeadCTLesionDetection/datasets/hemorrhage/labels.csv`。
2. 将 CT 影像统一放入 `HeadCTLesionDetection/datasets/hemorrhage/images/`。
3. 确认 `labels.csv` 中的 `image_path` 都能正确指向影像文件。
4. 运行 `train.py` 或 `train_monai.py` 生成 `best.pt`。
5. 运行 `evaluate.py` 输出测试集 `metrics.json` 和 `predictions.csv`。
6. 如果已有 voxel 级病灶 mask，再运行 `segmentation_prepare.py` 和 `train_nnunet.py` 做 nnU-Net 分割实验。
7. 用 `LESION_MODE=model` 启动 LesionDetection。
8. 用 `LESION_SERVICE_ENABLED=true` 启动 Orchestrator，完成真实模型模式联调。

阶段 K 的平台开发可以与真实病灶模型训练并行进行，但进入临床试运行前，仍必须使用正式数据完成 checkpoint 训练、独立测试集评估和医生复核验收。`smoke_best.pt` 只能用于技术联调，不能作为真实诊断模型发布。
