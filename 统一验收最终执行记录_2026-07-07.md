# 统一验收最终执行记录（2026-07-07）

## 一、验收标准复述

本轮验收严格按 `统一验收标准.md` 执行，不弱化为“服务能启动”或“脚本能跑”。验收标准包括：

- 开始前检查并覆盖远端 `master` 已有业务能力，界面风格向远端基准统一。
- 使用数据库真实数据和项目真实服务链路，不以 mock、fallback、smoke 结果冒充正式链路。
- 前后端接口、字段、状态枚举、页面展示必须一一匹配，前端数据必须来自后端数据库或真实服务。
- 患者、医生、管理员、医技、药房、头部 CT AI、报告、EMR 等模块必须按真实业务时序联动。
- AI 影像识别、RAG/LLM/记忆、SSE 流式输出必须给出结构化、专业、可读、可验证证据。
- 所有界面和交互必须有截图级证据，截图来自用户点击路径，不用手输隐藏 URL 代替用户入口。
- 收费、退费、药品库存必须专项验收，不能被整体端到端脚本笼统覆盖。
- 所有验收脚本和文档输出必须使用 UTF-8，不得写入乱码。
- 若某条标准没有证据，必须写“未覆盖”，不能写完成或通过。

## 五、正常用户入口路径

以下入口均要求从登录后的角色界面点击到达，不以手输隐藏 URL 作为通过证据：

- 患者端：患者登录 -> 首页/挂号/缴费/检查预约/报告/AI 问诊/我的。
- 医生端：医生登录 -> 待诊队列 -> 接诊 -> 病历 -> 开检查/检验/处方 -> AI 诊断 -> 头部 CT 报告工作台。
- 管理员端：管理员登录 -> 首页 -> 排班/AI 排班/医生科室/财务/药品库存。
- 医技端：医技登录 -> 检查/检验执行工作台 -> 任务执行/结果录入。
- 药房端：药师登录 -> 药房工作台 -> 库存/发药/退药/流水。
- 头部 CT 报告：医生端入口进入主平台内嵌报告工作台，报告服务独立地址仅用于健康检查和服务调试。
